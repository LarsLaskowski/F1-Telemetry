# Integrate F1 2026 (UDP packet format 2026)

Implementation plan for adding support for the **F1 26** game (`m_packetFormat = 2026`,
`m_gameYear = 26`) to the F1-Telemetry solution.

This plan is written so that one or more agents can execute it in order. It is based on a
field-by-field comparison of:

- `docs/F1 26 Telemetry Output Structures - v1.txt` (new C++ spec)
- `docs/F1 25 Telemetry Output Structures - v3.txt` (current C++ spec)
- the appendices in `docs/F1 2026 Telemetry.pdf` (Driver / Team / Track / Nationality IDs)

> **Status of sample data:** F1 2026 sample packets now **exist** under
> `F1Server.Tests/SampleData/F1-2026-*.packet`. They were generated by transforming the existing
> F1 2025 captures into the 2026 wire format (header re-stamped to `format = 2026`, `gameYear = 26`;
> 22 → 24 cars; all field-level layout changes applied). Unit tests for every packet that has a
> sample can therefore be **fully implemented now**. The only packets still without a sample are
> `CarTelemetry2` (brand-new, nothing to derive it from) and `FinalClassification`, `SessionHistory`,
> `TimeTrial`, `LapPositions` (no 2025 capture existed to convert) — see §4 for exactly which tests
> are implementable vs deferred, and the exact synthetic values baked into the new 2026-only fields.

## 0. Scope decisions (confirmed with product owner)

| Topic | Decision |
|---|---|
| **New `CarTelemetry2` packet (id 16)** | **Full implementation + persistence** — parse, expose live, and persist to a new entity/table with EF migrations for all three providers. |
| **New fields on already-processed packets** (CarStatus `ersHarvestLimitPerLap`; Session active-aero/DRS zones, start reaction time, extra assists) | **Parse + persist** — add to entities with EF migrations for MsSql / MySql / PostgreSql. |
| **Changed-but-unprocessed packets** (LobbyInfo, CarSetups, CarDamage, Motion, MotionEx, TyreSets) | **Keep as `UnknownData`** — only make sure the 24-car size/format does not break header/length handling. No 2026 data classes for these. |

## 1. Summary of format changes 2025 → 2026

### 1.1 Global / header
- `cs_maxNumCarsInUDPData`: **22 → 24**. This enlarges **every** per-car packet (Motion, LapData,
  Participants, CarSetups, CarTelemetry, CarStatus, FinalClassification, LobbyInfo, CarDamage,
  LapPositions, CarTelemetry2). All hard-coded 2026 packet sizes must use 24.
- `m_packetFormat = 2026`, `m_gameYear = 26`.
- **Header layout is unchanged** (29 bytes, same as 2023–2025). The raw `m_packetId` → enum mapping
  in `ReceivedPacketData.AnalyzePacketHeader` already does `packetType + 1`, so no header parsing
  change is needed beyond recognising the new packet id (see §1.3).
- New constants: `cs_maxActiveAeroZonesPerLap = 8`, `cs_maxDRSZonesPerLap = 4`.
- `cs_maxMarshalsZonePerLap` renamed to `cs_maxMarshalZonesPerLap` (value still 21 — cosmetic only).

### 1.2 New packet: `CarTelemetry2` (`ePacketIdCarTelemetry2 = 16`, 269 bytes)
Per-car struct `CarTelemetry2Data`:

| Field | Type | Notes |
|---|---|---|
| `m_activeAeroMode` | uint8 | 0 = Corner mode, 1 = Straight mode |
| `m_activeAeroAvailable` | uint8 | 0 = not available, 1 = available |
| `m_activeAeroActivationDistance` | uint16 | 0 = N/A, else available in X m |
| `m_overtakeAvailable` | uint8 | 0/1 |
| `m_overtakeActive` | uint8 | 0/1 |
| `m_overtakeActivationDistance` | uint16 | 0 = N/A, else available in X m |
| `m_2026Regulations` | uint8 | 0 = pre-2026, 1 = 2026 regs applicable |
| `m_drivingWrongWay` | uint8 | wrong-way flag |

Packet = header + `CarTelemetry2Data[24]`.

### 1.3 `PacketId` enum (game side)
`ePacketIdCarTelemetry2 = 16` is appended after `ePacketIdLapPositions = 15`. In our
`PacketTypes` enum (which is offset by +1) this becomes a new value **`CarTelemetry2 = 17`** after
`LapPositions = 16`.

### 1.4 Changed packets (field-level)

**Motion** (`CarMotionData`) — *unprocessed, stays `UnknownData`; informational only*
- `m_gForceLateral / Longitudinal / Vertical`: **float → int16** (quantised, divide by 1000.0f).
  Motion struct shrinks; total packet 1349 → 1325.

**Session** (`PacketSessionData`) — **processed, persist new fields**
- `m_formula` gains value `13 = F1 26`.
- New trailing fields (after `m_sector3LapDistanceStart`):
  - `m_activeAeroTrackStatus` (uint8: 0 = Full, 1 = Partial)
  - `m_numActiveAeroZonesFull` (uint8) + `ActiveAeroZone m_activeAeroZonesFull[8]`
  - `m_numActiveAeroZonesPartial` (uint8) + `ActiveAeroZone m_activeAeroZonesPartial[8]`
  - `m_numDRSZones` (uint8) + `DRSZone m_drsZones[4]`
  - `m_startReactionTime` (float, seconds; 0.0 if assisted start)
  - `m_antiLockBrakesAssist` (uint8: 0/1)
  - `m_tractionControlAssist` (uint8: 0 = off, 1 = medium, 2 = full)
  - `m_dynamicRacingLineHiVis` (uint8: 0/1)
  - `m_dynamicRacingLineColourBlind` (uint8: 0 = off, 1 = Protanopia, 2 = Deuteranopia, 3 = Tritanopia)
  - `m_recurringRewindPrompt` (uint8: 0/1)
  - New helper structs: `ActiveAeroZone { float zoneStart; float zoneEnd; }`,
    `DRSZone { float zoneStart; float zoneEnd; }`.
- Total 753 → 926 bytes.

**Lap** (`LapData`) — **processed; field layout unchanged**, only car count 22 → 24 (1285 → 1399).

**Event** — **processed; unchanged** (45 bytes). No new event codes.

**Participants** (`ParticipantData`) — **processed; breaking parse change**
- `m_driverId`: **uint8 → uint16** (network human now `65535`, was `255`).
- `m_networkId`: **uint8 → uint16**.
- `m_teamId`: **uint8 → uint16**.
- Total 1284 → 1470 bytes.

**CarSetups** — *unprocessed, stays `UnknownData`*; field layout unchanged, only car count (1133 → 1233).

**CarTelemetry** (`CarTelemetryData`) — **processed; breaking parse change**
- `m_engineTemperature`: **uint16 → uint8**.
- Total 1352 → 1448 bytes (24 cars, smaller engine-temp field).

**CarStatus** (`CarStatusData`) — **processed; persist new field**
- New field `m_ersHarvestLimitPerLap` (float) inserted **between** `m_ersHarvestedThisLapMGUH`
  and `m_ersDeployedThisLap`.
- `m_ersDeployMode` value `3` re-labelled `overtake → boost` (semantic only; enum/label update).
- Total 1239 → 1445 bytes.

**FinalClassification** — **processed; field layout unchanged**, only car count (1042 → 1134).

**LobbyInfo** — *unprocessed, stays `UnknownData`*
- `m_teamId`: uint8 → uint16 (65535 if no team). Informational only.

**CarDamage** — *unprocessed, stays `UnknownData`*; field layout unchanged, only car count.

**SessionHistory** — **processed; unchanged** (per-car packet, 1460 bytes, no size change).

**TyreSets** — *unprocessed, stays `UnknownData`*; unchanged (231 bytes).

**MotionEx** — *unprocessed, stays `UnknownData`*; unchanged (273 bytes).

**TimeTrial** (`TimeTrialDataSet`) — **processed; breaking parse change**
- `m_teamId`: uint8 → uint16. Total 101 → 104 bytes.

**LapPositions** — **processed; field layout unchanged**, array grows with 24 cars (1131 → 1231).

## 3. Backend implementation tasks

Work bottom-up: constants → enums → data classes/interfaces → parsers → entities/migrations →
processors → Web API/live → frontend → tests.

### 3.1 Constants — `F1Server.Core/Data/ConstData.cs`
Add a `#region Consts F1 2026` mirroring the 2025 region, with **recomputed sizes for 24 cars**:
- `F12026MaxCars = 24`, `F12026MaxLapPositions = 50`, `F12026MaxLaps = 100`,
  `F12026HeaderSize = 29`.
- `F12026SessionSize = 926`, `F12026LapSize = 57` / `F12026TotalLapSize = 57 * 24 = 1368`,
  `F12026EventSize` (= 2025 value, event unchanged), `F12026CarTelemetrySize` (per-car size with
  uint8 engine temp; recompute) and totals, `F12026ParticipantsSize` (uint16 ids → per-car larger),
  `F12026CarStatusSize` (per-car + `ersHarvestLimitPerLap`), `F12026SessionHistorySize` (unchanged
  layout, still 1460/“1431”-style — verify), `F12026FinalClassificationCarSize` /
  `F12026FinalClassificationSize`, `F12026LapPositionSize`, plus new
  `F12026CarTelemetry2Size = 269` and per-car `F12026CarTelemetry2CarSize = 11`.
- New shared sizes: `ActiveAeroZoneSize = 8`, `DrsZoneSize = 8`, and a 2026 driver-name length
  (still 32, reuse `DriverNameLength2025` or add `DriverNameLength2026 = 32`).

> Verify each computed size against the byte counts in the C++ header comments before trusting them.

### 3.2 Enum — `F1Server.Core/Enumerations/PacketTypes.cs`
Append `CarTelemetry2` (value 17) after `LapPositions`. Document it. No change needed in
`ReceivedPacketData.AnalyzePacketHeader` (the `+1` offset already maps raw id 16 → 17), but add a
unit test that raw id 16 resolves to `PacketTypes.CarTelemetry2`.

### 3.3 New data classes & interfaces — `F1Server.Core/Packets/Data/F1-2026/` and `.../Interfaces/F1-2026/`
Create a `F1-2026` folder pair mirroring `F1-2025`. For each **processed** packet, create a 2026
interface that **inherits the 2025 interface** and adds only the new members, plus a 2026 data
class implementing it. Required:

- `ICarStatus2026 / CarStatus2026`, `ICarStatusData2026 / CarStatusData2026` — add
  `ERSHarvestLimitPerLap` (float).
- `ICarTelemetry2026 / CarTelemetry2026`, `ICarTelemetryData2026 / CarTelemetryData2026` —
  `EngineTemperature` narrowed to a `byte`-sourced value (keep property type `ushort` for
  compatibility, just read a `uint8`).
- `ISessionData2026 / SessionData2026` — add active-aero zones, DRS zones, `StartReactionTime`,
  `AntiLockBrakesAssist`, `TractionControlAssist`, `DynamicRacingLineHiVis`,
  `DynamicRacingLineColourBlind`, `RecurringRewindPrompt`, `ActiveAeroTrackStatus`. Add supporting
  data types `ActiveAeroZone` and `DrsZone` under `Packets/Data/` (shared, like `MarshalZone`).
- `IParticipantData2026 / ParticipantData2026` — ids are already `ushort` in the 2025 data class;
  the change is in the **parser** (read uint16, see §3.4), so the 2026 interface may simply re-derive
  from 2025 unless a marker type is needed for the parser switch.
- `ITimeTrialData2026 / TimeTrialData2026`, `ITimeTrialDataSet2026 / TimeTrialDataSet2026` — `TeamId`
  is already `ushort`; change is parser-side (uint16).
- `ILapData2026 / LapData2026 / LapDataComplete2026`, `IFinalClassification2026 / *2026`,
  `ISessionHistory*2026`, `ILapPositions2026 / LapPositions2026`, `IEventDataDetails2026 /
  EventData2026 / EventDataDetails2026` — create even where the layout is unchanged, to keep the
  per-year switch pattern consistent (these can be thin `: I…2025` subtypes).
- **New:** `ICarTelemetry2Data2026 / CarTelemetry2Data2026` and a packet wrapper
  `CarTelemetry2_2026` (+ `ICarTelemetry2Base`, `ICarTelemetry2DataBase` base interfaces) holding
  `CarTelemetry2Data2026[24]`.

> Follow the existing inheritance chain (`I…Base` → `I…2019` → … → `I…2025`). Each new member needs
> XML docs (including private fields), Allman braces, `#region`s, and `== false`/`is null` idioms
> per `.claude/CLAUDE.md`.

### 3.4 Parsers — `F1Server.Core/Packets/PacketToObject/*`
For each processed packet, add a `2026 => new X(PacketHeader, new XData2026())` arm to the
`GameVersion switch`, and add `GameVersion >= 2026` guarded branches where fields changed:

- **`PacketToParticipants.cs`** — the most invasive. Add a `2026` arm in both switches. Add
  `ExtractParticipantDataSince2026` (or branch inside `ExtractParticipantBaseData`) that reads
  `DriverId`, `NetworkId`, `TeamId` as **uint16** instead of uint8 when `GameVersion >= 2026`.
  Update the human-driver / network-human checks: for 2026, network human id is **65535** (not 255);
  network id sentinel is also 65535.
- **`PacketToCarStatus.cs`** — add 2026 arm; when `GameVersion >= 2026` read the extra
  `ERSHarvestLimitPerLap` float between MGU-H harvested and `ERSDeployedThisLap`.
- **`PacketToCarTelemetry.cs`** — add 2026 arm; when `GameVersion >= 2026` read
  `EngineTemperature` as **uint8** (advance offset by 1 instead of 2).
- **`PacketToSessionData.cs`** — add 2026 arm; after the 2024/2025 trailing block, when
  `GameVersion >= 2026` read the active-aero status + zone arrays, DRS zone array, start reaction
  time and the five new assist/UI flags. Reuse a zone-array reader for `ActiveAeroZone[8]` (full and
  partial) and `DrsZone[4]`.
- **`PacketToTimeTrialData.cs`** — add 2026 arm; read `TeamId` as uint16 when `GameVersion >= 2026`.
- **`PacketToLapData.cs`, `PacketToFinalClassification.cs`, `PacketToSessionHistoryData.cs`,
  `PacketToLapPositions.cs`, `PacketToEventData.cs`** — add a `2026` arm; layout identical to 2025
  (only car-count/const differences flow through). Confirm any `MaxCars`/loop bounds read from the
  2026 consts, not hard-coded 22.
- **New `PacketToCarTelemetry2.cs`** — new transformer producing `CarTelemetry2_2026`. Register it.
- **`PacketToXBase.GetHeaderSize`** — add `2026 => ConstData.F12026HeaderSize` (29).
- **`PacketToXBase.AdjustSessionType`** — no change required (2026 session types match the 2024+
  mapping); add a regression test to confirm.
